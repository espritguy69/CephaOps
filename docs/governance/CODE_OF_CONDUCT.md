
# CephasOps Code of Conduct (Engineering)

## Engineering Expectations
- Write clean, documented, consistent code.
- Follow Clean Architecture.
- Respect multi-company security.
- Do not bypass validations.
- Always write tests for new features.

## Collaboration Rules
- No unreviewed commits to main.
- Use PR templates.
- Keep documentation updated.

## Anti-Pattern Rules
- No inline SQL.
- No business logic in controllers.
- No skipping audit fields.
- No unbounded queries without pagination.
